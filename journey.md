# Journey

ทุกบรรทัดโค้ดที่ผมเขียนวันนี้ ล้วนมีที่มาจากบทเรียนในแต่ละช่วงของชีวิต  
นี่คือเรื่องราวของผมตั้งแต่ความอยากรู้แรกในวัยเด็ก จนถึงการเป็น Full-Stack Developer ที่สร้างระบบให้คนใช้งานจริงในวันนี้

---

## 🧒 จุดเริ่มต้น — ลื้อคอม เพื่อเข้าใจระบบ

ตั้งแต่เด็ก ผมชอบแกะดูว่าสิ่งต่าง ๆ รอบตัวทำงานยังไง  
พี่ชายชอบชวนผมเล่นเกมทายกัน "แคชเชียร์รับเงินก่อน หรือเปิดลิ้นชักก่อน" หรือ "เครื่องกดน้ำทำงานยังไง"

และสิ่งที่ทำบ่อยที่สุดคือ **ลื้อคอมพี่ชาย** เพื่อดูว่าข้างในทำงานยังไง  
จริงๆมันก็อยากให้ผมลื้อแหละ เพื่อที่มันจะได้ไปเล่นร้านเกม
จนทำให้ผมซ่อมคอม ประกอบเครื่อง วิเคราะห์อาการเสียได้ตั้งแต่ยังเป็นเด็ก

> ความอยากรู้ว่า "ระบบทำงานยังไงข้างใน" คือสิ่งที่ผลักดันผมมาจนถึงทุกวันนี้

---

## 🎓 ฝึกงาน ปวช. — ครูคนแรกของชีวิต

ตอนฝึกงาน ปวช. ผมได้เข้าไปฝึกที่แผนก IT ของบริษัทอสังหาฯ แห่งหนึ่ง  
เป็นจุดที่ผมเริ่มเขียนโปรแกรมจริงจังเป็นครั้งแรก ด้วย **ASP.NET**

ที่นั่น ผมได้ทำงานประกบคู่กับ **วิศวกรคอมพิวเตอร์ระดับปริญญาโท** ที่สอนผมแบบไม่กั๊ก  
สอนทั้งวิธีคิดโครงสร้าง Database, โครงสร้างโปรแกรม, และวิธีคิดเวลาเจอปัญหา

พี่เขาเคยพูดประโยคหนึ่งที่ผมจำมาตลอด:

> "ถ้าเราเรียนกับใคร เราก็จะเหมือนคนนั้นแหละ"

ประโยคนี้เปลี่ยนวิธีเรียนรู้ของผมไปตลอดกาล  
ทำให้ผมตัดสินใจว่า — **จะเรียนรู้จากคนที่เก่งที่สุดเท่านั้น**

---

## 💥 อายุ 19–20 — บทเรียนจากความมั่นใจที่ไม่มีฐาน

หลังเรียนจบ ปวช. ผมตัดสินใจ **ไม่เรียนต่อ**  
เพราะความมั่นใจในตัวเองที่คิดว่า — "ผมทำเองได้"

### ครั้งที่ 1: FiveM Server

ผมเริ่มทำ FiveM Server กับพี่ชายและทีม ด้วยความมั่นใจเต็มที่  
แต่ความจริงคือ ผมเขียนโค้ดด้วยการ **ก็อปแล้วแก้ ไม่ได้เข้าใจจริง ๆ** ว่าฟังก์ชันรับค่าอะไร ส่งค่าอะไร  
พอเจอปัญหาที่แก้ไม่ได้ ก็ทำต่อไม่ออก สุดท้ายเจ๊ง

### พลิกบทบาท: Crypto Bomb & NFT Game Bot

หลังจากนั้น ผมได้ร่วมทำงานกับทีมในยุค **Crypto Bomb / NFT Game** ที่กำลังบูม  
หัวหน้าทีมคือ **CEO ของบริษัทน้ำดื่มแห่งหนึ่ง** ผู้ที่มีประสบการณ์ทำธุรกิจจริง

ผมรับบทบาทเขียน **Auto-click Bot** สำหรับเกม NFT  
และร่วมลงทุนกับพี่ชายในวงเงิน **50,000 บาท** เพื่อเริ่มสร้างผลตอบแทน

ตลอดช่วงนั้น เราทำกำไรร่วมกันได้กว่า **100,000 บาท** — กำไรกว่า 100% จากเงินลงทุน  
แต่สิ่งที่มีค่ามากกว่ากำไร คือ **บทเรียนทางธุรกิจ** ที่ผมได้รับจากพี่หัวหน้าทีม:

- **การวางโครงสร้างธุรกิจ** และการแบ่งสัดส่วนหุ้นส่วนที่เป็นธรรม
- **การทำ Roadmap** ที่มี milestone ชัดเจน
- **การตัดสินใจในภาวะวิกฤต** — สิ่งที่ผมจดจำมากที่สุด
  ในช่วงปลายของยุค NFT เริ่มมีโปรเจคที่ **Rug Pull** กันเยอะ  
  พี่หัวหน้าทีมตัดสินใจอย่างเด็ดขาดว่า — **"ขายเหรียญออกทั้งหมด ไม่ต้องสนใจราคา"**

หลังจากนั้นไม่นาน มูลค่าเหรียญตกลงจนแทบไม่มีค่าเหลือ  
ในวันที่ทุกคนยังเชื่อในกระแส — ทีมเราออกได้ทันด้วยการตัดสินใจของคนที่มีประสบการณ์

> นั่นคือครั้งแรกที่ผมได้เห็นว่า — การ **"รู้ว่าเมื่อไหร่ควรหยุด"** มีค่าพอๆกับ **"รู้ว่าเมื่อไหร่ควรเริ่ม"**

ประสบการณ์ครั้งนี้ทำให้ผมเริ่มสนใจ "ธุรกิจ" ในมุมที่ลึกกว่าการเขียนโค้ด —  
ไม่ใช่แค่การหาเงิน แต่คือการเข้าใจ **timing, risk management, และ decision-making**

### ครั้งที่ 2: FiveM Server (อีกครั้ง)

ทีมเดิมจาก Crypto Bomb มาลงทุนทำ FiveM Server ต่อ คราวนี้เป็นทางการมากขึ้น  
แต่พอมี **Software Engineer ตัวจริง** เข้ามาในทีม ผมถึงรู้ว่า  
ที่ผ่านมา ผมแทบไม่เข้าใจอะไรเลย — และทีมก็แตกอีกครั้ง

> ความมั่นใจที่ไม่มีความเข้าใจรองรับ มีแต่จะทำให้เสียหาย

### กลับสู่โลกความจริง — Customer Service

หลังจากทุกอย่างจบลง ความฝันที่จะทำธุรกิจเองพักไว้ก่อน  
ผมต้องหางานประจำทำ — และได้เริ่มต้นใหม่ในตำแหน่ง **Customer Service** ที่โรงงานสติกเกอร์แห่งหนึ่ง

ตอนนั้นผมไม่ได้ศึกษาเรื่องเขียนโค้ดเลย เพราะยังไม่มีแรงจูงใจ  
แต่สิ่งที่ผมเก่งมาตั้งแต่เด็กยังอยู่ — **ฮาร์ดแวร์**  
ประกอบคอม, ซ่อมเครื่อง, จัดสเปค, เช็คอาการ ทำได้สบายและมั่นใจมาก

ระหว่างทำ CS ผมเริ่มเขียน **VBA ใน Excel** เพื่อช่วยให้งานในแผนกเร็วขึ้น  
เริ่มจากใช้เอง จนหัวหน้าแผนกเห็นและเอาไปใช้ต่อ

วันหนึ่ง บริษัทมีการย้ายออฟฟิศ ผมไปช่วยทีม IT ขนคอม  
ผู้บริหารเห็นว่าผมทำงานคล่อง ชอบคอม และเขียน VBA ได้  
แม้ตำแหน่ง IT จะยังไม่ว่าง — เขาก็เปิดโอกาสให้ผม

ตอนนั้นบริษัทใช้ **FileMaker Pro** เป็นระบบหลัก  
ผู้บริหารเรียก **ที่ปรึกษาภายนอก** เข้ามาสอนทีม IT ให้พัฒนาระบบเองได้  
และเขาให้ผมเข้าไปเรียนด้วย

> นี่คือจุดที่ผมเริ่มเห็นว่า — โลกของ "Internal System" ในองค์กรจริงเป็นยังไง

ประสบการณ์ครั้งนี้สำคัญมาก เพราะมันเป็นเหตุผลที่ — หลายปีต่อมา — ผู้บริหารคนเดิมโทรกลับมาเสนองาน IT ให้ผมอีกครั้ง

## 🏭 อายุ 21 — จุดเปลี่ยนที่ Wann

หลังออกจากโรงงานสติกเกอร์ ผมได้เข้ามาที่ **Wann Cosmetics** ในตำแหน่ง IT Support

วันแรกที่สัมภาษณ์ ผู้บริหารถามผมตรง ๆ ว่า — **"เขียนโปรแกรมได้ไหม?"**  
นั่นคือสัญญาณที่บอกผมว่า ที่นี่ **เปิดโอกาสให้ Developer**

ในช่วงทดลองงาน ผมพิสูจน์ตัวเองด้วยสิ่งที่ผมแข็งที่สุด —  
จัดการระบบ Hardware ในองค์กร ซ่อมและจัดสเปคเครื่อง ตอบปัญหา IT ของพนักงานได้รวดเร็ว  
รวมถึงเรียนรู้ระบบงานของบริษัทได้เร็วกว่ามาตรฐาน

ทำให้ผม **ผ่านโปรในเวลา 1.5 เดือน** จากมาตรฐาน 3 เดือน

หลังจากผ่านโปร ผมเริ่มสังเกตว่า **ระบบเอกสารในแผนกยังใช้ Microsoft Access**  
ผมจึงขออนุญาตหัวหน้าเสนอเปลี่ยนระบบ — ทำ Flowchart, ออกแบบ Database, และเลือก MERN Stack ที่ทันสมัย  
หัวหน้าไฟเขียว ผมจึงขอเวลาศึกษา Stack 1 เดือน

โปรเจคแรกที่ผมเขียนคือ **Realtime Chat ตามคลิปฝรั่ง**  
คลิปยาว 7 ชั่วโมง — ผมใช้เวลา 2 อาทิตย์ เพราะหยุดทุกครั้งที่ไม่เข้าใจ และไปขุด **why** ก่อนเดินต่อ

นี่คือครั้งแรกที่ผมเรียนเขียนโค้ดอย่างจริงจัง โดยเลือกเรียนจาก **Software Engineer ตัวจริง** จากต่างประเทศ  
ไม่ใช่จาก tutorial ผิว ๆ — เพราะผมจำคำของพี่ที่เคยสอนได้ว่า  
"ถ้าเราเรียนกับใคร เราก็จะเหมือนคนนั้นแหละ"

ผมมีทุนเดิมอยู่: พื้นฐาน **โครงสร้าง Database** ที่ได้จากตอนฝึกงาน ปวช.  
และความเข้าใจ **Internal System** จากประสบการณ์ FileMaker Pro ที่ผ่านมา

ทุกวันหลังเลิกงาน 18:00 ผมเรียนต่อที่บ้านจนถึง 23:00  
ทั้งหมดนั้น ผมทำในขณะที่ตำแหน่งจริงคือ **IT Support** และไม่มีใครสั่งให้ทำ

ผลลัพธ์คือ:

- ขยับเป็น **IT Specialist** ภายใน 2 ปี
- ได้รับ **รางวัลพนักงานดีเด่น**
- สร้างระบบ Internal ที่ทุกแผนกในบริษัทใช้งานจริงทุกวัน

## 🤝 โอกาสที่เกือบเปลี่ยนชีวิต — SAP Interface Project

ปีแรกที่ Wann เราขึ้นระบบ SAP เป็นครั้งแรก  
ในจังหวะที่ทีม Vendor เข้ามาลงระบบใหม่หลังเซิร์ฟเวอร์โดนไวรัส  
ผมได้แลกเปลี่ยนความรู้กับทีมเขา และเขาเห็นว่าผมเขียนเว็บได้  
จึงชวนผมเข้าทีมพัฒนา **Interface เชื่อมต่อ SAP** ในส่วน WMS + QC

ผมรับโอกาสนั้น แม้หัวหน้าจะเตือนว่าเสี่ยง  
เพราะผมเห็นโอกาสในการได้เรียนรู้ "ไส้ใน" ของ ERP ระดับองค์กร

ตลอด 6 เดือน ผมทำงานร่วมกับ Backend ส่งข้อมูลกันผ่าน API พร้อม JSON Docs  
จนได้รับการสนับสนุนจาก CEO ของ Vendor ที่ตั้งใจจะให้เราเป็นบริษัทลูก

แต่แล้วทุกอย่างก็จบลงในวันที่ **หัวหน้าทีมจากไปอย่างกระทันหัน**  
โปรเจคนั้นจบลงที่ตรงนั้น

> สิ่งที่ผมเสียดายไม่ใช่โปรเจค — แต่คือคนดี ๆ ที่ผมได้รู้จัก

แต่บทเรียนและประสบการณ์เรื่อง **SAP Integration**, การทำงานกับทีมระดับมืออาชีพ  
และความกล้าที่จะเดิมพันกับโอกาส — ทั้งหมดยังอยู่กับผมจนถึงวันนี้

## 🚀 ปัจจุบัน — Full-Stack Developer ที่เชื่อใน "ปัญหาจริง"

ตลอด 4 ปีที่ Wann ผมรับผิดชอบ **Internal System** แบบ End-to-End  
ตั้งแต่ออกแบบ Database, พัฒนา Backend ด้วย Express.js / NestJS, Frontend ด้วย React / Next.js  
ไปจนถึงพัฒนา **Satellite System** เชื่อมต่อ SAP Business One  
และ Integrate Hardware (Flow Meter, เครื่องชั่ง) ผ่าน Serial Communication

ในโครงการล่าสุด ผมรับบทบาท **Project Coordinator** ในโปรเจค WannSmart  
ทำงานร่วมกับทีมพัฒนาภายนอก ตั้งแต่เก็บ Requirement → ออกแบบ Flow → ส่งต่อทีม Backend → ติดตามจนส่งมอบ

วันนี้ผมมั่นใจว่า:

> **Developer ที่ดี ไม่ใช่แค่คนที่ Code เก่ง — แต่ต้องเข้าใจ "ปัญหาจริง" ที่ต้องการแก้ไขด้วย**

ตอนนี้ผมพร้อมที่จะก้าวผ่านตัวเองไปอีกขั้น — อยากเข้าไปอยู่ในทีมที่เก่ง ที่ร่วมกันสร้างสิ่งที่ยิ่งใหญ่ได้
ผมอยากใช้ความเข้าใจเรื่อง ERP และ ธุรกิจ ที่สะสมมา เพื่อร่วมส่งมอบ Product ที่ใช้งานง่าย แก้ปัญหาได้จริง และมีต้นทุนที่เหมาะสม — เพื่อประโยชน์สูงสุดของทั้งทีมและลูกค้า
